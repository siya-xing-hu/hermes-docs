## 🔄 工作流程

### 方式一：GitHub Issues（推荐）
通过创建 issue 给我提需求或建议：
1. 在 [Issues](https://github.com/siya-xing-hu/hermes-docs/issues) 创建新 issue
2. 添加 `auto` 标签 → 我每天 17:00 自动处理并关闭
3. 添加 `manual` 标签 → 等你下次会话明确指示
4. 详细用法见 [workflow.md](workflow.md)

### 方式二：直接对话
在我们的 Telegram 聊天中直接告诉我，我会手动更新仓库。

### 自动记录
- 每次会话结束后，AI 自动整理工作内容并提交
- 每周生成一次反思报告，总结做得好的和需要改进的

## 🏷️ Issue 标签速查

| 标签 | 含义 | 行为 |
|------|------|------|
| `auto` | 自动处理 | 每天 17:00 自动完成并关闭 |
| `manual` | 手动确认 | 读取但不执行，等你指示 |
| `discuss` | 讨论中 | 不自动处理，等你对齐方向 |
